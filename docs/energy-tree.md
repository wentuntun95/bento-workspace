## 核心技术

- **HTML5 Canvas API**，无任何第三方动画库
- `requestAnimationFrame` 驱动每帧动画
- 几何数据模块加载时预计算缓存（`const GEO_*`），不在每帧重算

**坐标系：**
```
原点 (ox, oy) = (W*0.52, H*0.94)   ← 树干底部
U = 7 (CSS px per pixel block)
gx, gy = 网格坐标（负 gy = 向上）
```

## 树冠算法（最终方案）

### 单椭圆公式 + 噪声扰动
```
椭圆参数：ECX=0, ECY=-27, ERX=19, ERY=18
边界噪声：NOISE_THR=0.18（±18% 半径范围内随机包含/排除）
```

```typescript
const r = Math.sqrt((dx*dx)/(ERX*ERX) + (dy*dy)/(ERY*ERY));
// r < 1-thr → 确定在内
// r > 1+thr → 确定在外
// 边界区域：随机包含，概率 = 1 - (r-(1-thr))/(2*thr)
```

> ⚠️ 为什么不用多 blob 叠加：多椭圆交界处会产生不可预测的突起，单椭圆在数学上保证圆形轮廓，只有边界区域有有机锯齿。

## 果实数据来源（Hybrid B）

```typescript
// 本周 taskHistory 展开 + 今日 tasks.completed 实时叠加
allFruits = buildWeeklyFruits(taskHistory, tasks);
```

- **历史部分**：`taskHistory` 中 ISO 周号匹配当前周的条目，展开各类型数量
- **今日部分**：`tasks.filter(completed)` 实时追加
- **周一清零**：`getISOWeekFromStr(entry.date)` 不匹配当前周则跳过
- **最大显示**：50 个果实槽（`FRUIT_SLOTS`）

## 版本迭代历史

| 版本 | 关键变化 | 问题 |
|------|---------|------|
| v1–v3 | 基础像素树，Y轴明暗 | 平面感强 |
| v4 | 实心格子+Y轴渐变，削减树干 | 无体积感 |
| v5 | 分叉枝臂+叶球簇 | 树干与树冠断口 |
| v6 | 大主椭圆骨架消除断口 | 形状扁平（飞碟） |
| v7 | 细胞噪声叶片质感 | 侧翼突起，树冠三角 |
| v8 | **单椭圆公式**+定向光照叶簇 | ✅ 当前最终版 |

**核心教训**：像素艺术树冠的「有机感」来自**边界区域的随机采样**，而不是多个元素叠加。单椭圆公式在数学上保证整体形状可控，边界噪声产生自然锯齿，两者分离职责是 v8 成功的关键。
